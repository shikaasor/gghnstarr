<task type="auto">
  <name>Task 1: Create app/sitemap.ts and app/robots.ts</name>
  <files>app/sitemap.ts, app/robots.ts</files>
  <action>
    Create two new files using Next.js 16 built-in file conventions. These generate static sitemap.xml and robots.txt at build time with output:'export'. No external packages needed.

    CREATE app/sitemap.ts:
    ```typescript
    import type { MetadataRoute } from 'next';
    import { getAllBriefs } from '@/lib/content';

    const BASE_URL = process.env.NEXT_PUBLIC_SITE_URL || 'https://gghnstarr.vercel.app';

    export default function sitemap(): MetadataRoute.Sitemap {
      const briefs = getAllBriefs();

      const briefEntries: MetadataRoute.Sitemap = briefs.map((b) => ({
        url: `${BASE_URL}/briefs/${b.slug}`,
        lastModified: new Date(b.publicationDate),
        changeFrequency: 'monthly' as const,
        priority: 0.8,
      }));

      return [
        {
          url: BASE_URL,
          lastModified: new Date(),
          changeFrequency: 'weekly' as const,
          priority: 1,
        },
        {
          url: `${BASE_URL}/briefs`,
          lastModified: new Date(),
          changeFrequency: 'weekly' as const,
          priority: 0.9,
        },
        {
          url: `${BASE_URL}/methodology`,
          changeFrequency: 'monthly' as const,
          priority: 0.6,
        },
        {
          url: `${BASE_URL}/experts`,
          changeFrequency: 'monthly' as const,
          priority: 0.6,
        },
        {
          url: `${BASE_URL}/contact`,
          changeFrequency: 'yearly' as const,
          priority: 0.4,
        },
        ...briefEntries,
      ];
    }
    ```

    CREATE app/robots.ts:
    ```typescript
    import type { MetadataRoute } from 'next';

    const BASE_URL = process.env.NEXT_PUBLIC_SITE_URL || 'https://gghnstarr.vercel.app';

    export default function robots(): MetadataRoute.Robots {
      return {
        rules: {
          userAgent: '*',
          allow: '/',
        },
        sitemap: `${BASE_URL}/sitemap.xml`,
      };
    }
    ```

    WHY built-in over next-sitemap: The built-in app/sitemap.ts and app/robots.ts file conventions work natively with output:'export' — they generate static files at build time. The next-sitemap package adds an external dependency and requires postbuild configuration. For a site with fewer than 20 URLs, the built-in approach is sufficient and zero-dependency.

    WHY reuse getAllBriefs(): The existing lib/content.ts already handles path.join(process.cwd(), 'content', 'briefs-index.json') correctly. Reusing it avoids duplicating file-reading logic and prevents path resolution bugs (a common pitfall documented in RESEARCH.md Pitfall 4).

    WHY 'as const' on changeFrequency: TypeScript requires the literal union type for MetadataRoute.Sitemap changeFrequency field. Without 'as const', TypeScript infers 'string' which doesn't match the expected type.

<task type="auto">
  <name>Task 2: Add @media print block to globals.css</name>
  <files>app/globals.css</files>
  <action>
    Append a @media print block at the end of app/globals.css after the existing @theme block. This provides print-friendly output for all pages, especially brief detail pages.

    APPEND to the end of app/globals.css (after the @custom-variant dark line):

    ```css
    /* ==========================================================================
       Print Styles — Brief Detail Pages
       Target: Elements on /briefs/[slug] pages for clean printed output
       ========================================================================== */

    @media print {
      /* A4 page setup with readable margins */
      @page {
        margin: 2cm;
        size: A4 portrait;
      }

      /* Reset backgrounds and shadows for print */
      *,
      *::before,
      *::after {
        background: transparent !important;
        color: #000 !important;
        box-shadow: none !important;
        text-shadow: none !important;
      }

      body {
        font-size: 12pt;
        line-height: 1.6;
        font-family: Georgia, serif;
      }

      /* Hide site chrome: header, footer, navigation */
      header,
      footer,
      nav {
        display: none !important;
      }

      /* Hide elements marked with .no-print utility class
         Applied in 05-01 to: download buttons div, prev/next nav */
      .no-print {
        display: none !important;
      }

      /* Hide all download/action buttons */
      button {
        display: none !important;
      }

      /* Preserve heading-to-content flow across page breaks */
      h1,
      h2,
      h3,
      h4 {
        break-after: avoid;
        page-break-after: avoid;
      }

      /* Keep content sections intact — no mid-section page breaks */
      section,
      li,
      p,
      blockquote {
        break-inside: avoid;
        page-break-inside: avoid;
      }

      /* Constrain thumbnail image for print layout */
      img {
        max-width: 180px !important;
        height: auto !important;
        page-break-inside: avoid;
      }

      /* Show absolute link URLs inline (useful for references) */
      a[href^="http"]::after {
        content: " (" attr(href) ")";
        font-size: 10pt;
        color: #444 !important;
      }

      /* But suppress URL display for internal and anchor links */
      a[href^="/"]::after,
      a[href^="#"]::after {
        content: "";
      }
    }
    ```

    WHY !important on display:none: Tailwind v4 utility classes may have higher CSS specificity. Using !important ensures print styles override utility class display properties. This is consistent with the RESEARCH.md recommendation (Pitfall 5).

    WHY target header/footer/nav elements directly: These are semantic HTML elements used by the site's Header and Footer components. Tailwind class names are unpredictable in build output — semantic element selectors are reliable.

    WHY .no-print class: The prev/next nav and download buttons on the brief detail page were given .no-print in 05-01 Task 2. This CSS block makes that class functional.

    WHY Georgia for print body font: The system font stack (ui-sans-serif, system-ui) may not be available or optimal in print contexts. Georgia is a universally available serif font that produces readable printed text.

    DO NOT modify any existing lines in globals.css above this block — the @theme block, @import "tailwindcss", and @custom-variant dark lines must remain untouched.
  </action>
