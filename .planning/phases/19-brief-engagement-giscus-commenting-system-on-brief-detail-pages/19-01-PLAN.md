    Create `app/components/briefs/GiscusComments.tsx` with the following:

    ```tsx
    'use client';

    import Giscus from '@giscus/react';

    export function GiscusComments() {
      return (
        <section className="no-print mt-12 pt-8 border-t border-slate-200">
          <h2 className="font-serif text-xl text-navy-950 font-bold mb-6">Discussion</h2>
          <Giscus
            repo="shikaasor/gghnstarr"
            repoId="PLACEHOLDER_REPO_ID"
            category="General"
            categoryId="PLACEHOLDER_CATEGORY_ID"
            mapping="pathname"
            reactionsEnabled="1"
            emitMetadata="0"
            inputPosition="top"
            theme="light"
            lang="en"
            loading="lazy"
          />
        </section>
      );
    }
    ```

    Key decisions (per research):
    - `'use client'` directive required — Giscus renders an iframe that depends on browser APIs
    - `mapping="pathname"` — auto-creates one GitHub Discussion per brief URL slug (no manual mapping needed)
    - `theme="light"` — no dark mode on site; do NOT use "preferred_color_scheme"
    - `loading="lazy"` — defers iframe load until user scrolls to section
    - `reactionsEnabled="1"` — allows emoji reactions on discussions
    - `emitMetadata="0"` — no postMessage overhead needed
    - `inputPosition="top"` — comment box above existing comments (more accessible)
    - The `no-print` class on `<section>` hides comments when printing (globals.css already defines `.no-print { display: none !important; }`)
    - Place outside `max-w-3xl` content column — the section is a direct child of Container, same level as Prev/Next nav
    - `repoId` and `categoryId` are PLACEHOLDER strings — user must fill in real values from giscus.app before Phase 19 is considered fully live; they are NOT secrets (hardcode, not env vars)
    - The "Discussion" heading uses the same `font-serif text-xl text-navy-950 font-bold` pattern used throughout the brief detail page
    - Border and spacing match the Prev/Next nav section for visual consistency
  </action>
