<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Human visual verification of /tools-directory</name>
  <what-built>
    A searchable, filterable /tools-directory page rendering 50 One Health tools as cards, with Organization Level / Audience Type / Scope chip filters and a free-text search box, plus a "Tools" link in the site header.
  </what-built>
  <how-to-verify>
    1. Run `npm run dev` and open http://localhost:3000/tools-directory
    2. Confirm the teal hero banner renders with the "Tools Directory" badge and heading, and that a grid of tool cards appears below (result count should read 50 items, or 12 visible with pagination if PAGE_SIZE applies).
    3. Type a term into the search box (e.g. "surveillance" or "WHO") — confirm the grid narrows to matching tools and the result count updates.
    4. Click chips in each filter dimension (Organization Level, Audience Type, Scope) — confirm the grid narrows; confirm combining chips across dimensions ANDs correctly and chips within a dimension OR; confirm "Clear filters" resets everything including the search box.
    5. Click a tool card title that has a link — confirm it opens the external site in a new tab. Confirm a tool with no URL renders its name as plain (non-link) text.
    6. Confirm cards with long descriptions are clamped to ~3 lines (no runaway card height).
    7. In the header, confirm a "Tools" link is present and navigates to /tools-directory. Check desktop nav spacing with 11 links is acceptable (gap-6); resize to mobile width and confirm the hamburger menu lists "Tools".
  </how-to-verify>
  <resume-signal>Type "approved" or describe any issues (rendering, filter behavior, broken links, nav spacing) to fix.</resume-signal>
