<task type="auto">
  <name>Task 2: Scraper script scripts/fetch-news.mjs</name>
  <files>scripts/fetch-news.mjs</files>
  <action>
    Create scripts/fetch-news.mjs as an ES module (.mjs). The script:

    **Imports:**
    - `import fs from 'fs'` and `import path from 'path'` and `import { fileURLToPath } from 'url'`
    - `import { XMLParser } from 'fast-xml-parser'` — resolves from repo root node_modules

    **Constants:**
    - `__dirname = path.dirname(fileURLToPath(import.meta.url))`
    - `NEWS_JSON_PATH = path.join(__dirname, '../content/news.json')`

    **`loadExisting()`:** Try `JSON.parse(fs.readFileSync(NEWS_JSON_PATH, 'utf-8'))`, catch returns `[]`.

    **`normalizePubDate(pubdate)`:** Parse `"2026 May"` → `"2026-05-01"`, `"2026 Apr 15"` → `"2026-04-15"`, `"2026"` → `"2026-01-01"`. Month map: `{Jan:1, Feb:2, Mar:3, Apr:4, May:5, Jun:6, Jul:7, Aug:8, Sep:9, Oct:10, Nov:11, Dec:12}`.

    **`fetchArxiv()`:**
    - Query: `ti:"antimicrobial resistance" OR ti:"antibiotic resistance" OR abs:"antimicrobial resistance" OR abs:"antibiotic resistance"` — encodeURIComponent
    - URL: `https://export.arxiv.org/api/query?search_query=${query}&sortBy=submittedDate&sortOrder=descending&max_results=100`
    - Parse with `new XMLParser({ ignoreAttributes: false, attributeNamePrefix: '@_' })`
    - `entries = parsed?.feed?.entry ?? []`; ensure array with `Array.isArray(entries) ? entries : [entries]`
    - Per entry: extract `arxivId` from `entry.id.split('/abs/')[1].replace(/v\d+$/, '')`, `authors` from `entry.author` array (handle single vs array), `publishedDate = entry.published.slice(0, 10)` (use `<published>` NOT `<updated>`), `journal = entry['arxiv:primary_category']?.['@_term'] ?? ''`
    - Return array of NewsArticle objects with `source: 'arXiv'`, `url: \`https://arxiv.org/abs/\${arxivId}\``

    **`fetchPubmed()`:**
    - Step 1 ESearch: `https://eutils.ncbi.nlm.nih.gov/entrez/eutils/esearch.fcgi?db=pubmed&term=antimicrobial+resistance[tiab]+OR+antibiotic+resistance[tiab]&datetype=pdat&reldate=7&retmax=200&retmode=json&usehistory=y${apiParam}`
    - If `parseInt(searchData.esearchresult.count) === 0` return `[]`
    - Step 2 ESummary (JSON): `https://eutils.ncbi.nlm.nih.gov/entrez/eutils/esummary.fcgi?db=pubmed&query_key=${query_key}&WebEnv=${WebEnv}&retmode=json${apiParam}`
    - Step 3 EFetch for abstracts (XML): `https://eutils.ncbi.nlm.nih.gov/entrez/eutils/efetch.fcgi?db=pubmed&query_key=${query_key}&WebEnv=${WebEnv}&retmode=xml&rettype=abstract${apiParam}` — parse with XMLParser, build PMID→abstract map from `//PubmedArticle/MedlineCitation/Article/Abstract/AbstractText`
    - Build results: `results.uids.map(uid => ...)`. From ESummary: `r.title`, `r.authors[0].name` for first author, `r.source` for journal, `r.pubdate` → `normalizePubDate()`. From DOI: `r.articleids?.find(id => id.idtype === 'doi')?.value`. Authors string: `authors.length > 1 ? \`\${firstAuthor} et al.\` : firstAuthor`
    - Return array with `source: 'PubMed'`, `url: \`https://pubmed.ncbi.nlm.nih.gov/\${uid}/\``
    - Wrap all fetch calls in try/catch; on error log warning and return `[]` (silent fallback to existing news.json is acceptable per user decisions)

    **`deduplicateMerged(articles)`:**
    - Input: `[...pubmedArticles, ...arxivArticles]` — PubMed first so PubMed wins on duplicate
    - `normalizeTitle(t) = t.toLowerCase().replace(/\s+/g, ' ').trim()`
    - `normalizeDoi(doi) = doi ? doi.toLowerCase().trim() : null`
    - Use a Map: key = `normalizeDoi(article.doi) ?? normalizeTitle(article.title)`. Only insert if key not seen.
    - Return `Array.from(seen.values())`

    **`writeNews(articles)`:**
    - Sort by `publishedDate` descending (string compare works for YYYY-MM-DD)
    - Slice to 200
    - `fs.writeFileSync(NEWS_JSON_PATH, JSON.stringify(sorted, null, 2), 'utf-8')`
    - `console.log(\`Wrote \${sorted.length} articles to news.json\`)`

    **Main execution:**
    ```javascript
    const existing = loadExisting();
    const [pubmedArticles, arxivArticles] = await Promise.all([fetchPubmed(), fetchArxiv()]);
    const merged = deduplicateMerged([...pubmedArticles, ...arxivArticles, ...existing]);
    writeNews(merged);
    ```

    **`.github/workflows/fetch-news.yml`:**
    Create `.github/` and `.github/workflows/` directories if they don't exist, then create the workflow file:

    ```yaml
    name: Fetch AMR News

    on:
      schedule:
        - cron: '0 6 * * *'   # 06:00 UTC daily
      workflow_dispatch:        # Allow manual trigger for testing

    permissions:
      contents: write           # Required to push commits back to main

    jobs:
      fetch-news:
        runs-on: ubuntu-latest
        steps:
          - name: Checkout repository
            uses: actions/checkout@v4

          - name: Set up Node.js
            uses: actions/setup-node@v4
            with:
              node-version: '20'

          - name: Install dependencies
            run: npm ci

          - name: Run news scraper
            env:
              NCBI_API_KEY: ${{ secrets.NCBI_API_KEY }}
            run: node scripts/fetch-news.mjs

          - name: Commit and push news.json
            run: |
              git config user.name "github-actions[bot]"
              git config user.email "github-actions[bot]@users.noreply.github.com"
              git add content/news.json
              git diff --staged --quiet && echo "No new articles — skipping commit" || \
                git commit -m "chore(news): daily article refresh $(date -u +%Y-%m-%d)"
              git push
    ```

    Key decisions in this workflow:
    - `npm ci` at root installs fast-xml-parser (it's in devDependencies of root package.json)
    - `git diff --staged --quiet && ... || git commit` prevents empty commit error when no new articles
    - No Vercel deploy hook needed — pushing to main auto-triggers Vercel rebuild
    - NCBI_API_KEY is optional (no API key = 3 req/s limit, sufficient for daily runs); user must add secret in GitHub repo settings when ready
  </action>
