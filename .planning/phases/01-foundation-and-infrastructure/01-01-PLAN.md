<task type="auto">
  <name>Task 1: Configure Git LFS for binary assets</name>
  <files>.gitattributes</files>
  <action>
    Run the following commands from the project root to initialize Git LFS and configure tracking rules for binary assets. This MUST happen before any PDF or image files are added to the repository — once a binary is committed without LFS, removing it requires rewriting git history.

    ```bash
    git lfs install
    git lfs track "*.pdf"
    git lfs track "*.jpg"
    git lfs track "*.png"
    ```

    These commands automatically update `.gitattributes`. Verify the file contains all three rules, then stage and commit it:

    ```bash
    git add .gitattributes
    git commit -m "chore: configure Git LFS for binary assets (pdf, jpg, png)"
    ```

    Expected `.gitattributes` contents after running the above:
    ```
    *.pdf filter=lfs diff=lfs merge=lfs -text
    *.jpg filter=lfs diff=lfs merge=lfs -text
    *.png filter=lfs diff=lfs merge=lfs -text
    ```

    NOTE: If `git lfs install` fails with "command not found", Git LFS must be installed on the machine first (https://git-lfs.github.com/). Do not proceed until `git lfs status` runs without error.
  </action>
  <verify>
    Run `git lfs status` — should report no errors.
    Run `cat .gitattributes` — must show all three filter=lfs lines.
    Run `git log --oneline -1` — should show the LFS configuration commit.
  </verify>
  <done>`.gitattributes` exists and is committed with filter=lfs rules for *.pdf, *.jpg, and *.png. `git lfs status` runs without error.</done>

<task type="auto">
  <name>Task 2: Scaffold Next.js project, configure static export, install Tailwind v4, set up brand tokens and dark mode</name>

  <action>
    **Step A: Scaffold the project**

    Run from the project root (the directory already has git initialized — scaffold in-place):
    ```bash
    npx create-next-app@latest . --typescript --eslint --app --no-tailwind --no-src-dir --import-alias "@/*"
    ```

    Use `--no-tailwind` to avoid installing Tailwind v3 by default — we install v4 manually next.
    If prompted about existing files (package.json, README, etc.), choose to overwrite.

    **Step B: Install Tailwind v4 and UI utilities**
    ```bash
    npm install tailwindcss@4.2.2 @tailwindcss/postcss@4.2.2
    npm install clsx@2.1.1 tailwind-merge@3.5.0 lucide-react@1.6.0
    ```

    **Step C: Configure static export**

    Replace the generated `next.config.js` with:
    ```javascript
    /** @type {import('next').NextConfig} */
    const nextConfig = {
      output: 'export',
      images: {
        unoptimized: true,  // Required: default loader requires a server
      },
    };

    module.exports = nextConfig;
    ```

    Do NOT add `trailingSlash: true` — the default (false) works correctly with Vercel.

    **Step D: Create PostCSS config for Tailwind v4**

    Create `postcss.config.mjs` at project root (replacing any existing postcss.config.js):
    ```javascript
    const config = {
      plugins: {
        "@tailwindcss/postcss": {},
      },
    };
    export default config;
    ```

    NOTE: Tailwind v4 requires `@tailwindcss/postcss` as the plugin — NOT `tailwindcss` (that was v3). Using the wrong plugin name results in Tailwind classes not being processed (site renders with no styles, no error).

    **Step E: Configure globals.css with brand tokens and dark mode**

    Replace the generated `app/globals.css` with the following. Do NOT create `tailwind.config.js` — Tailwind v4 uses CSS-first config exclusively:
    ```css
    @import "tailwindcss";

    /* GGHN STARR Brand Tokens — Navy/Teal Design Spec */
    /* Colors derived from GGHN STARR Design Spec: Deep Navy + Medical Teal */
    @theme {
      /* Primary palette */
      --color-navy-950: #0F172A;      /* Deep Navy — primary dark background */
      --color-navy-900: #1E293B;      /* Navy — card backgrounds, secondary dark */
      --color-navy-800: #2D3F55;      /* Navy lighter — borders, dividers */
      --color-teal-600: #0D9488;      /* Medical Teal — CTAs, accent, links */
      --color-teal-500: #14B8A6;      /* Teal lighter — hover states */
      --color-teal-400: #2DD4BF;      /* Teal light — highlights, dark mode accent */

      /* Neutral palette */
      --color-slate-50: #F8FAFC;
      --color-slate-100: #F1F5F9;
      --color-slate-200: #E2E8F0;
      --color-slate-400: #94A3B8;
      --color-slate-600: #475569;
      --color-slate-900: #0F172A;

      /* System font stacks — NO external fonts (hard requirement for bandwidth) */
      /* Headings use serif for authoritative policy look; body uses sans */
      --font-sans: ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont,
                   "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif;
      --font-serif: Georgia, Cambria, "Times New Roman", Times, serif;
    }

    /* Class-based dark mode — toggle .dark on <html> with JavaScript */
    /* Tailwind v4 requires @custom-variant; the v3 darkMode: 'class' config has no effect */
    @custom-variant dark (&:where(.dark, .dark *));
    ```

    **Step F: Update app/layout.tsx**

    Replace the generated root layout with:
    ```tsx
    import type { Metadata } from 'next';
    import './globals.css';

    export const metadata: Metadata = {
      title: {
        default: 'GGHN STARR | AMR Policy Intelligence for Africa',
        template: '%s | GGHN STARR',
      },
      description: 'Evidence-backed AMR policy briefs for African health decision-makers.',
    };

    export default function RootLayout({
      children,
    }: {
      children: React.ReactNode;
    }) {
      return (
        <html lang="en" suppressHydrationWarning>
          <body className="bg-slate-50 dark:bg-navy-950 font-sans antialiased">
            {children}
          </body>
        </html>
      );
    }
    ```

    NOTE: `suppressHydrationWarning` is required on `<html>` — the dark mode toggle will add/remove the `dark` class client-side, which differs from server-rendered HTML and causes React hydration mismatch warnings without this attribute.

    The Header and Footer components will be imported here in Plan 03 once they are built. Leave `{children}` unwrapped for now.

    **Step G: Verify static build**

    ```bash
    npm run build
    ```

    Must complete without errors. The `out/` directory must be created. If you see any errors about server components, API routes, or image optimization — fix before proceeding.

    **Step H: Connect to Vercel (checkpoint:human-action delegated to user)**

    This step requires human browser interaction — document it clearly for the user:

    1. Push current state to GitHub main branch:
       ```bash
       git add -A && git commit -m "feat(01): scaffold Next.js 16 with Tailwind v4 static export" && git push origin main
       ```
    2. Go to https://vercel.com/new and import the GitHub repository
    3. In project settings, leave all defaults — Vercel auto-detects Next.js
    4. After first deploy completes, go to Project Settings > Git > "Ignored Build Step" and add:
       ```
       if [[ "$VERCEL_GIT_COMMIT_REF" == "main" ]] ; then exit 1 ; else exit 0 ; fi
       ```
       This configures main-branch-only deployment (exit 0 cancels build for non-main branches).
    5. In Project Settings > Environment Variables, add:
       - Name: `NEXT_PUBLIC_SITE_URL`
       - Value: `https://your-project.vercel.app` (use the actual subdomain Vercel assigned)
       - Environment: Production
    6. Trigger a redeploy from Vercel dashboard to apply the env var.

    Record the Vercel URL in `.planning/STATE.md` under a "Deployment" section.
